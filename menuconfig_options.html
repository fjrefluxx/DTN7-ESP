Use Bundle ID's for reception Confirmation

If enabled, the local node stores a set of received bundle ID hashes. The hashes of the IDs of all received bundles are inserted into this set, which is emptied during each BPoL advertisement. Additionally, each node in the forwarded-to list of stored bundles has a set of bundle ID hashes that is used to check if the last transmission of the bundle to the node was successful. Enabling this affects: EpidemicRouter::checkForwardedTo(), encodeAdvertisePacket(), DTN7::bundleReceiver(), Node (Class), Node::serialize(), BundleProtocolAgent::bundleReception(). Only compatible with EpidemicRouter and BPoL compatible LoRa CLA. For more information see main readme "experimental features".


default False
